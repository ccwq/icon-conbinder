# 领域文档

当 engineering skills 探索代码库时，应按下面的规则读取本仓库的领域文档。

## 探索前先读这些

- 仓库根目录下的 `CONTEXT.md`，如果存在
- 与当前工作区域相关的 `docs/adr/` 中的 ADR

如果这些文件不存在，就静默跳过，不要因为缺失而阻塞，也不要先要求创建。

## 文件结构

单上下文仓库：

```
/
├── CONTEXT.md
├── docs/adr/
└── src/
```

## 使用术语表中的词汇

当输出里提到领域概念时，要使用 `CONTEXT.md` 中定义的术语，不要改用术语表明确避免的同义词。

如果需要的概念还没有写进术语表，这通常表示项目还缺一个正式术语，或者需要在 `/grill-with-docs` 里补充说明。

## 标记 ADR 冲突

如果输出内容和现有 ADR 冲突，要明确指出，而不是悄悄覆盖。
